# 홈 화면 리디자인 (급여 카드 화이트화 + 폴리시)

**날짜:** 2026-07-09
**대상 파일:** `src/screens/HomeScreen.tsx`, `src/theme/colors.ts`

## 목표

타겟 이미지에 맞춰 홈 화면을 리디자인한다. 현재 구현은 구조적으로 이미 이미지와
거의 동일하므로 **비주얼만** 조정한다. 로직·스토어·캘린더 계산·모달은 손대지 않는다.
테마 토큰 기반으로 작업해 다크모드(`system/light/dark`, 이미 동작)가 깨지지 않게 한다.

## 변경 사항

### 1. 예상 급여 카드 (핵심)
- 배경: `colors.brand`(파랑) → `colors.surfaceElevated` (라이트=흰색 / 다크=#2c2c2e)
- 장식용 blob(`blob1`, `blob2`) 제거
- 레이아웃 재구성:
  - 상단행: `{month}월 예상 급여` + 눈 아이콘 (좌) · `근무 N일 · N시간` (우)
  - 하단행: `₩1,284,000` 큰 숫자 (좌, `colors.brand`) · `↑ 지난달 +N%` pill (우, 하단 정렬)
- 라벨/메타 텍스트: `colors.textSecondary`
- 눈 아이콘·트렌드 chevron: `colors.textSecondary` / `colors.brand`

### 2. 트렌드 pill
- 배경: 반투명 흰색 → `colors.brandSoft`
- 글씨/아이콘: `colors.brand`
- 신규 토큰 `brandSoft`: 라이트 `#e8edff`, 다크 `rgba(96,165,250,0.16)`

### 3. 카드 소프트 섀도우
- 급여·캘린더·일정 카드에 공통 그림자 스타일 적용
  (iOS `shadowColor/Opacity/Radius/Offset` + Android `elevation`)
- 다크모드에선 자연히 약하게 보임 (별도 분기 불필요)

### 4. 월 이동 화살표
- 유지. 스타일만 이미지 톤에 맞게 정리 (`colors.surface` 배경 유지).

## 변경하지 않는 것
캘린더 그리드/점/금액 표기, 범례, 상세 카드 구조, FAB, 광고배너, 모든 계산 로직,
모달, 스토어, 훅.

## 검증
- 앱 실행(또는 타입체크) 후 라이트/다크 양쪽에서 급여 카드가 흰색/다크 표면으로
  렌더되고 숫자가 브랜드 컬러로 표시되는지 확인.
- 기존 테스트(`jest`) 통과 확인 — 이 변경은 스타일만이라 로직 테스트에 영향 없어야 함.
